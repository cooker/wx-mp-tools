# 工具导航 · 设计系统（初稿）

基于 UI UX Pro Max 匹配结果，结合 **Bento Grid + Minimalism + Dark Mode** 风格。

---

## 1. 设计方向

| 维度 | 选择 | 说明 |
|------|------|------|
| 主风格 | Bento Grid | 模块化卡片、非对称网格、多种尺寸、Apple 风 |
| 副风格 | Minimalism | 极简、留白多、高对比、几何感、无冗余 |
| 主题 | Dark Mode | 深色背景、护眼、高可读 |

---

## 2. 色彩系统

**来源：** Dark Mode (OLED) + Minimalism 单色系

| 角色 | Hex | 用途 |
|------|-----|------|
| 背景 | `#000000` | 主背景（深黑） |
| 背景提升 | `#121212` | 卡片、悬浮层 |
| 背景次级 | `#0A0E27` | 区块区分、Midnight Blue |
| 边框 | `#1E293B` | 分隔线、卡片边框 |
| 正文 | `#E6EDF3` / `#F1F5F9` | 主文字 |
| 次要文字 | `#8B949E` | 描述、辅助信息 |
| 强调色 | `#0080FF` | 链接、hover、焦点（Electric Blue） |
| 强调色备用 | `#F59E0B` | 可选（琥珀，与现有风格接近） |

**原则：** 单主色（蓝或琥珀），其余以灰阶为主，避免多色杂乱。

---

## 3. 布局与 Bento Grid

### 3.1 网格结构

- **主内容区：** CSS Grid，非对称 `grid-template-columns`，支持 `span` 实现不同宽度
- **卡片尺寸：** 小（1x1）、中（2x1）、大（2x2），按内容重要度分配
- **间距：** `gap: 1rem`（16px）或 `1.5rem`（24px），保持节奏感
- **圆角：** `border-radius: 12px` 或 `16px`（rounded-xl）

### 3.2 模块划分

```
┌─────────────────────────────────────────────────────────┐
│  Header（标题 + 描述）                                    │
├───────────────────────────────┬─────────────────────────┤
│  Bento 主网格                 │  右侧边栏                 │
│  ┌──────┬──────┬──────────┐  │  ┌─────────────────────┐ │
│  │ 小   │ 小   │  大      │  │  │ 公告                 │ │
│  ├──────┼──────┤          │  │  │ （sticky）           │ │
│  │ 小   │ 中   │          │  │  └─────────────────────┘ │
│  └──────┴──────┴──────────┘  │                         │
├───────────────────────────────┴─────────────────────────┤
│  底部广告位                                              │
└─────────────────────────────────────────────────────────┘
```

- **分类块：** 每类为一个 bento 卡片，可 `span` 不同列宽
- **工具卡片：** 统一小卡片，网格内均分
- **公告：** 右侧固定列宽（约 280px），`position: sticky`

## 4. 排版

| 用途 | 字体 | 字号 | 字重 | 备注 |
|------|------|------|------|------|
| 站点标题 | Outfit / Inter | 1.75rem–2.25rem | 700 | 响应式 clamp |
| 分类标题 | 同正文 | 0.95rem | 600 | 可选 uppercase + letter-spacing |
| 工具名称 | 同正文 | 0.95rem | 500 | 卡片主文字 |
| 描述/次要 | 同正文 | 0.85rem | 400 | 次要信息 |
| 正文 | Outfit / Inter | 1rem | 400 | line-height 1.5–1.6 |

**原则：** 保持极简，单字族或 Heading + Body 搭配，避免花哨字体。

---

## 5. 动效与交互

| 类型 | 时长 | 缓动 | 说明 |
|------|------|------|------|
| Hover | 200–250ms | ease | 卡片、链接、按钮 |
| 入场 | 400–500ms | ease-out | 区块淡入、错峰 stagger |
| Hover 缩放 | scale(1.02) | 200ms | 卡片轻微放大，不破坏布局 |

**原则：** 克制使用动效，以提升反馈为主；支持 `prefers-reduced-motion: reduce`。

---

## 6. 组件规范

### 6.1 工具卡片（Bento 格）

- 背景：`#121212`，边框 `1px solid #1E293B`
- 圆角：`12px`
- 内边距：`1rem`
- Hover：`scale(1.02)`、边框高亮（强调色）、`box-shadow` 轻微增强
- 光标：`cursor: pointer`
- 无障碍：`focus-visible` 可见轮廓

### 6.2 分类区块

- 可作为 bento 大格，内部再网格排布工具卡片
- 分类标题：小号、大写或加 letter-spacing，与内容区分

### 6.3 公告卡片

- 背景：`#121212`，边框同工具卡片
- 可 `sticky`，随滚动保持可见
- 列表项可点击，hover 使用强调色

### 6.4 广告位

- 与主风格一致，避免过于花哨
- 边框、圆角与工具卡片统一

---

## 7. 避免事项（Anti-patterns）

- ❌ 使用 emoji 作图标，建议 SVG（Heroicons、Lucide）
- ❌ 过度动效、复杂过渡
- ❌ 多种强调色混用
- ❌ 浅色背景（与深色主题冲突）
- ❌ 过小点击区域（建议 ≥ 44×44px）

---

## 8. 响应式断点

| 断点 | 布局 |
|------|------|
| &lt; 768px | 单列，公告上移到主内容上方 |
| 768px–1024px | 双列，主内容 + 侧边栏 |
| &gt; 1024px | 完整 Bento + 侧边栏，max-width 约 1200px |

---

## 9. 实施优先级

1. **P0：** 色彩变量、Bento 网格结构、工具卡片样式
2. **P1：** 公告区、广告位与整体统一
3. **P2：** 动效、stagger 入场、`prefers-reduced-motion`
