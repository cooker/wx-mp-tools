# SQL 模板目录

约定见仓库根目录 `SQL.md`。

- 将 `.md` 放在本目录（当前实现为**单层**清单：根下每个业务一个 `xxx.md`，id 为文件名不含 `.md`）。
- **`manifest.json` 勿手改**：由 `scripts/generate-sql-manifest.js` 扫描本目录自动生成；`pnpm dev` / `pnpm build` 前会自动执行。开发中若服务已在跑，可执行 `pnpm run sync:sql-templates` 后刷新页面。

## Front matter（必填）

`code`、`name`、`bizType`、`version`、`description`

## 分节标题

使用二级标题，且须与解析器一致（全大写）：

`## DDL` / `SEED` / `QUERY` / `UPDATE` / `REPORT` / `DEV`（按需使用，顺序不限）

## 小节结构

每个小节：

1. `### 标题`（列表与编辑器中显示）
2. 可选 `> NOTE: 说明`
3. 单个 ` ```sql ` … ` ``` ` 代码块（块内为合法 SQL 或 SQL 注释形态的开发说明）

## 页面展示约定（与 `SqlTemplatesView.vue` 一致）

| 分节 | 展示方式 |
|------|----------|
| **DDL** | 全部片段纵向展示；顶部标签仅锚点滚动；每段 CodeMirror |
| **DEV** | 全部片段纵向展示；顶部标签锚点；**正文由 `--` / `/* */` 转成 Markdown 渲染**（表格、标题、列表、围栏代码） |
| **其余** | 单片段切换 + CodeMirror；支持格式化 / 复制 |

## DEV 分节编写建议（便于 Markdown 阅读）

- 优先使用**行注释** `--`：去掉 `--` 后的文本即 Markdown 源码。
- 使用二级/三级标题时写在注释里，例如：`-- ## 接口一览`、`-- ### 1. 要点`（渲染为标题）。
- **表格**：行以 `| 列1 | 列2 |` 形式书写；首行表头下一行数据行即可（构建时会自动补 GFM 分隔行）。
- **JSON 示例**：用围栏包裹，每行仍加 `--` 前缀，例如：`-- ```json`、对象行、最后 `-- ``` `。
- 段落之间用**空一行**（注释里写 `--` 空行或纯 `--`）。
- 块注释 `/* ... */` 也可：块内行首 `*` 会被去掉；适合大段说明。

## 其它分节

小节内：`### 标题`，可选 `> NOTE:`，再接单个 ` ```sql ` 代码块（可含 `--` SQL 注释）。
